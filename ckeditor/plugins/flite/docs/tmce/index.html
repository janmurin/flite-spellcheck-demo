Requirements

  • IMPORTANT: FLITE requires jQuery
  • FLITE has been tested on Firefox 15+, Chrome 13+, Safari 10.x and Edge. Support for MSIE 9+ is not guaranteed, although the current version seems to work on it.

Main Features

  • Enable / disable track changes mode
  • Show / hide tracked changes
  • Accept/Reject all/some changes
  • Display customizable tooltips tracked changes

Getting Started

  • Add the flite plugin to TinyMCE. The simplest way to do this is by adding the following line to your editor configuration block: 
    plugins: "flite"

  • Optionally include flite-interface.js in your source, so you can use the various constants defined in it rather than string literals.

    <script type="text/javascript" src="tinymce/plugins/flite/js/flite-interface.js"></script>

Configuration

The FLITE plugin is automatically activated after you install as described above. For the full details of tweaking the loading process, toolbar commands, users and more, see the API documentation for full details.

API

The FLITE plugin exposes a wide range of methods, events and properties for controlling and reporting the tracked changes. See the API documentation for details.

In order to interact with a FLITE plugin instance, you need to obtain a reference to one.

The simplest way to do this is through a fully initialized TinyMCE instance, e.g. myEditor.plugins.flite.

You can also get a reference to a FLITE plugin instance by listening to the event FLITE.Events.INIT (or "flite:init" in string literal form) fired by the editor instance. For example:


	tinymce.init({
		selector: "my-editor",
		// various options
		flite: {
			// various FLITE options
		},
		setup: function(ed) {
			ed.on(FLITE.Events.INIT /* "flite:init" */, function(event) {
				var flite = event.flite;
				// do something with this instance of FLITE
			});
		}
	});

Notes

  • The FLITE tracking engine adds markup to the editor content. All the change tracking data is stored inline, as part of the document.
  • FLITE tracks a wide range of editing actions, but not all of them. Most notably, style edits and search/replace operations are not tracked (they're on our roadmap, though).
